## Public Release Rule

A private-development repository may become public only after:

1. no restricted microdata are tracked;
2. no local paths are present;
3. outputs are aggregate, synthetic or legally redistributable;
4. README clearly states the public/private boundary;
5. validation confirms no forbidden files;
6. the user explicitly approves publication.
